Sprites

Using sprites is a common technique to serve images that are meant to be used as icons. It means combining several images into one image and then using positioning to position them on an appropriate element. Pros of using this technique is (in most cases) bandwidth saving and making fewer requests to the server, thus saving even more since concurrent requests are usually limited. Cons are that a background image can't be centered on a large element since this would show other images along the edges. It is also important to decide which images should be packed together. Packing 1500 images in a sprite if you're only going to use one or two is not good since it would take a long time to get the initial download (it would hopefully be clien side cached after that).

Creating a spritesheet is fairly easy, if you have a predefined element size that should never change, then you can simply stack your images horizontally and vertically in an image with enough margin to prevent other images to show along the edges. If you know that some of your elements size might increase in one direction, then you're better of creating a spritsheet that stacks all the images either horizontally or vertically, thus avoiding the problem with neighboring images.

Responsive

Sprites

The easiest way to make sprites responsive is to create several spritesheets in varying sizes and change the image used for all your sprites (easiest way is to use a colon separated selector that defines the background-image). You should always give your smaller sprites as much room as your larger ones. Lets say that your larger sprites are 128x128 pixels and your smaller ones are 64x64 pixels and that you are using horizontally stacked sprites. your second sprite should then be position at y128 and x0, the third one should be at y256 and x0. This is to avoid having to change the background-position of every single element in a media-query, we just have to change the background-image

This example will obiously look weird on sizes other than desktop large since it displays the different spritesheets without swapping them.

.microsoft,
.wp,
.facebook {
	background-image: url('../images/icons/tiles.png');
	background-repeat: no-repeat;
}

.microsoft {background-position:0px 0px}
.wp{background-position: 0px -128px;}
.facebook{background-position:0px -256px;}

@media (min-width:768px) and (max-width: 979px){
	.microsoft,
	.wp,
	.facebook {
		background-image: url('../images/icons/tiles-large.png');
	}
}

@media (min-width:1200px){
	.microsoft,
	.wp,
	.facebook {
		background-image: url('../images/icons/tiles-tablet.png');
	}
}

The advantages of this method gets clearer the more sprites you add as the overhead per sprite gets smaller.
