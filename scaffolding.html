Grid

Having a great grid system is essential to building a great and maintainable website, responsive or not. A grid doesn't have to implement 12 or 16 or 24 columns. If all you need is a website with pages that can contain content and a side menu, then why implement the rest? Just build a span12, span9 and span3 class, the main thing is that your grid is well defined in its boundries, this allows you to easily implement other sizes later on. You could of course call your classes span1, span3 and span4 or anything else, but this makes it harder to extend and they might lose their semantic values.

Containers

M web UI uses containers as a way to structure content content. Containers and rows in other grid system are often just a simple block element meant to separate content vertically. MWUI adds to the semantic value of a container by allowing it to also structure content horizontally, this is done by combining the .container with a .span*. This makes it immediately clear that this is a container that covers * number of columns on the page, and all of its children have a different kind of bond than if the hierarchy were .container > .span > element. It also allows for extremely flexible layouts where containers can contain other containers instead of having to add a separate span element in between. Containers are not meant to visaully structure content and therefore contains no padding or margins, that is up to a child span to decide.

Spans

Spans are the structuring elements closest to the content. It can be a block element such as a div that contains several elements. Or it can be a content element such as a h2 or a p element. What differs spans from containers are that they are meant to separate content on not just a basic layout level, but on a visual level too. This is why the spans have margins that make up the gutter of the grid. The MWUI grid is built on 24 columns and a span for covering any number of these columns, this is what creates the real layout. Since the MWUI grid is so flexible you can mix and match spans and containers to create the right layout and semantic for you, the spans inside of a container doesn't have to match up with the columns that the container covers, it will simple create a hole in your layout that may or may not be what you want.

Using the grid

The grid is flexible in the sense that any container can have containers as children and any element can be a span. This removes some unnecessary markup while preserving the semantic nature of the grid. A container can also have a span class to allow containers to be horizontally aligned to each other. This also allows for a higher grade of flexibility in responsive layouts that uses transforming spans and containers.

Example

A normal website layout is to have a layout wide header followed by navigation to the left, content in the middle, info to the right and a layout wide footer below.

Responsive

Responsive web design or RWD gives us the power to show any element on our page in a different way on different device sizes. This is extremely helpful when adding mobile and tablet friendly versions of our site. In most cases we can get by by only editing our stylesheets and making minor changes to our layout, providing that our page layout is well defined. However, if your mobile experience differs greatly from your desktop, then perhaps it's better to look at separating your codebase. This could still be done through new layouts/views and some responsive magic depending on your platform of choice.

MWUI Responsive

The easiest way to enable responsiveness in MWUI is to include the metro-responsive.css file. This includes not only containers and spans that adapt to different devices, but also transforming spans that enables you to easily create layouts that highlights information in different ways on different devise sizes.

Transforming spans

There is often a need for parts of a layout to cover different areas on different devices. This is why MWUI comes with "transforming" spans. These are spans that that act as larger or smaller spans on different devices. An element with the span20 and span16-tablet classes would for instance cover 20 columns on a desktop or large desktop computer, but 16 columns on a tablet.

Example

Take the earlier grid example, perhaps we want the right information column to be shown before the main content on a tablet to allow more horizontal size for the main content.

We also have to transform the content element to make use of the new space so we increase it from 12 columns on large and desktop to cover 18 columns on a tablet.
